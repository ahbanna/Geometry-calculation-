What is DOM and what is the purpose of DOM?

DOM stands for Document Object Model. By using it, we can create, change, or remove the document structure, style, and content. In order to make our page more dynamic, we can also add events to the elements. It represents the web page as a tree of objects or as a tree of nodes. A node represents an HTML element.

Purpose of DOM:

1. DOM can easily manipulate all HTML elements. Manipulation implies that we can add, change and remove all HTML elements. Apart from that, we can add styling, modify, and remove elements.

2. A DOM represents a document as nodes and objects that can be manipulated by programming languages.

3. We can modify the class, style, and other attributes of any HTML element.

4. It allows create applications that update the data of the page without needing a refresh.

How will you select HTML elements using DOM? Name the DOM methods?

There are many ways in which we can select HTML elements using the Document Object Model (DOM). Such as

  • getElementsByTagName()
  • getElementsByClassName()
  • getElementById()
  • querySelector()
  • querySelectorAll()

For Example:

Some common DOM methods are given below:

getElementById() : It selects an element by ID

getElementsByClassName() : It selects all elements that have a particular class name.

getElementsByTagName() : It returns an array of elements which have a specific tag name.

createElement() : it is used to create an element & place it anywhere in the DOM structure.

getAttribute() : It provides the value of any attribute of an element on a page.

setAttribute(): It sets the value of an attribute on an element.

innerHTML() : It sets or returns the HTML content.

appendChild() : It adds a new child node to an element.

removeChild(): It removes a node from an element.

What is event bubble?

In the DOM (Document Object Model), event bubbling occurs when an element receives an event, and that event is propagated through its ancestor and parent elements until reaching the root element.

With it, we can attach listeners to parent elements and handle events triggered on child elements without having to attach listeners to each child element separately.Sometime it is necessary to prevent one trigger on one element from causing multiple triggers on ancestors.

There are two methods to prevent event bubbling:

1. stopPropagation():
2. stopImmediatePropagation()

What is a callback function and why will you use it?

A callback function is an argument that is passed to another function so that it can be executed in that other function. It run after another function has finished.

There are several reasons for using callback functions. The main reason is asynchronous operations, such as fetching data from an API or reading file from a disk. It is often used to handle events, such as when a user clicks on a button or a timer expires.

As a result of callback functions, we are able to write code that is more versatile, modulatable, and efficient
